Blockchain

Wallet and token analysis, flow mapping, and entity labeling across chains.

Overview

Diverg's blockchain capabilities let you profile wallets, trace tokens and funds, and map counterparties. We support Ethereum and Solana today, with more chains and Layer 2s on the roadmap. When you configure API keys for Solscan Pro, Arkham, Etherscan, and Bubblemaps, you get richer labels and flow visualizations; without them, we still run analysis and clearly mark what was skipped. See Fact-Only Reporting.

Wallet & token analysis

For any address or token contract, Diverg can:

  • Profile the wallet: balance history, top tokens, and transaction patterns
  • Analyze token distribution, holder clustering, and large movements
  • Normalize transfers across tokens and chains for consistent reporting

With Arkham (and similar) integration, addresses can be labeled (e.g. exchange, known scam) so reports are easier to interpret.

Flow visualization

Fund flows are mapped into diagrams showing movement between wallets, mixers, CEXs, and bridges. Bubblemaps-style integration helps identify clustering and suspicious patterns. Outputs are suitable for incident response, due diligence, or evidence packaging.

Data sources

Blockchain data is pulled from:

  • Solscan Pro — Solana-focused data and labels
  • Arkham — Entity labeling and intelligence
  • Etherscan — Ethereum data and verification
  • Bubblemaps — Holder and flow visualization

Public RPC and block explorer data is used when API keys are not set; advanced features (e.g. full flow diagrams, labels) are then marked as skipped in the report.

Use cases

Typical applications: token launch and deployer due diligence, incident response (tracking attacker wallets and exchange deposits), rug-pull and centralization analysis, and compliance or legal evidence. See the Whitepaper for more.
